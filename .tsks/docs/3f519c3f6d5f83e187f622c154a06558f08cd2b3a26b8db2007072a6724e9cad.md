# Implementation Plan: Fix Game List Refresh and Add Live Game State Updates

**Issue ID**: lgm-b96
**Priority**: P2
**Type**: Feature
**Dependencies**: 
- lgm-j24 (Add WebSocket support)

## 1. Summary

Implement live game state updates that automatically refresh the game list and individual game views without manual polling, using the WebSocket infrastructure from lgm-j24.

## 2. Problem

Current game list has no live updates:

- **Stale Data**: Game list doesn't show new games or player changes
- **Manual Refresh**: Users must manually refresh to see updates
- **Poor UX**: Players miss games starting or joining
- **No Turn Notifications**: Players don't know when turns are processed

## 3. Proposed Solution

Use WebSocket events to update game state in real-time:

1. **Game List Updates**: Listen for game created/started/ended events
2. **Player Updates**: Listen for player joined/left events
3. **Turn Updates**: Listen for turn processed events
4. **UI Notifications**: Show toast notifications for important events

## 4. Implementation Plan

### Phase 1: WebSocket Events (2-3 hours)

Add events to WebSocketService:
```typescript
// Server events
- game:created
- game:started
- game:ended
- game:player_joined
- game:player_left
- game:turn_processed
```

### Phase 2: Update Games Store (2-3 hours)

```typescript
// client/src/stores/Games.store.ts
export const useGamesStore = defineStore('games', {
    actions: {
        handleGameCreated(game: any) {
            this.games.unshift(game);
        },
        
        handlePlayerJoined(event: any) {
            const game = this.games.find(g => g.id === event.gameId);
            if (game) {
                game.players.push(event.player);
            }
        },
        
        handleTurnProcessed(event: any) {
            const game = this.games.find(g => g.id === event.gameId);
            if (game) {
                game.turn = event.turn;
                game.world = event.world;
            }
        }
    }
});
```

### Phase 3: UI Components (2-3 hours)

Add notifications and visual indicators for:
- New games available
- Players joining/leaving
- Turns processed
- Game started/ended

### Phase 4: Testing (1-2 hours)

Test real-time updates with multiple clients.
