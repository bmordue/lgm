# Implementation Plan: Standardize API Response Formats

**Issue ID**: lgm-092
**Priority**: P2
**Type**: Task
**Dependencies**: None

## 1. Summary

Standardize all API response formats across the codebase to improve consistency, error handling, and client-side processing.

## 2. Problem

Current API responses are inconsistent:

- **Mixed Formats**: Some return objects, some return arrays
- **Inconsistent Errors**: Error responses vary by endpoint
- **No Envelope**: No standard response wrapper
- **Missing Metadata**: No pagination, timestamps, or request IDs

### Current State
```typescript
// Some endpoints return bare objects
{ id: 1, name: 'Game 1' }

// Others return wrapped responses
{ success: true, data: {...} }

// Errors are inconsistent
{ error: 'message' }
{ success: false, message: 'error' }
```

## 3. Proposed Solution

Implement standard response format:

```typescript
// Success Response
{
    success: true,
    data: T,
    meta: {
        requestId: string,
        timestamp: string,
        version: string
    }
}

// Error Response
{
    success: false,
    error: {
        code: string,
        message: string,
        details?: any
    },
    meta: {
        requestId: string,
        timestamp: string
    }
}

// Paginated Response
{
    success: true,
    data: T[],
    meta: {
        page: number,
        limit: number,
        total: number,
        totalPages: number
    }
}
```

## 4. Implementation Plan

### Phase 1: Define Response Types (1-2 hours)

Create `api/types/ApiResponse.ts`:
```typescript
export interface ApiResponse<T = any> {
    success: boolean;
    data?: T;
    error?: ApiError;
    meta: ResponseMeta;
}

export interface ApiError {
    code: string;
    message: string;
    details?: any;
}

export interface ResponseMeta {
    requestId: string;
    timestamp: string;
    version: string;
    pagination?: PaginationMeta;
}

export interface PaginationMeta {
    page: number;
    limit: number;
    total: number;
    totalPages: number;
}
```

### Phase 2: Create Response Helper (1-2 hours)

Create `api/utils/responseHelper.ts`:
```typescript
export function successResponse<T>(data: T, meta?: Partial<ResponseMeta>): ApiResponse<T> {
    return {
        success: true,
        data,
        meta: buildMeta(meta)
    };
}

export function errorResponse(code: string, message: string, details?: any): ApiResponse {
    return {
        success: false,
        error: { code, message, details },
        meta: buildMeta()
    };
}

function buildMeta(overrides?: Partial<ResponseMeta>): ResponseMeta {
    return {
        requestId: generateRequestId(),
        timestamp: new Date().toISOString(),
        version: '1.0',
        ...overrides
    };
}
```

### Phase 3: Update Controllers (3-4 hours)

Update all controllers to use response helpers:
```typescript
// Before
return game;

// After
return successResponse(game);
```

### Phase 4: Update Error Handling (2-3 hours)

Create standardized error handler middleware.

### Phase 5: Update Client (2-3 hours)

Update API client to handle standardized responses.
