# Implementation Plan: Add API Endpoint for Player-Specific World State

**Issue ID**: lgm-wb3
**Priority**: P3
**Type**: Task
**Dependencies**: None

## 1. Summary

Create a dedicated API endpoint to fetch the current game state for a specific player, including visibility-filtered world data, actor positions, and turn information.

## 2. Problem

Current system lacks a clean way to get player-specific world state:

- **No Refresh Endpoint**: Can't get full state after joining
- **Visibility Not Applied**: Raw world data exposed
- **Turn Results Only**: State only available after turn processing
- **Client Workarounds**: Frontend must piece together state from multiple sources

## 3. Proposed Solution

Add endpoint: `GET /games/{gameId}/players/{playerId}/state`

Returns visibility-filtered world state for the requesting player.

## 4. Implementation Plan

### Phase 1: Add API Spec (1 hour)

Update `api/spec/api.yml`:
```yaml
/games/{gameId}/players/{playerId}/state:
  get:
    summary: Get current game state for a player
    operationId: getPlayerGameState
    parameters:
      - name: gameId
        in: path
        required: true
        schema:
          type: integer
      - name: playerId
        in: path
        required: true
        schema:
          type: integer
    responses:
      "200":
        description: Player-specific game state
        content:
          application/json:
            schema:
              type: object
              properties:
                gameId:
                  type: integer
                playerId:
                  type: integer
                turn:
                  type: integer
                world:
                  $ref: '#/components/schemas/World'
                playerCount:
                  type: integer
```

### Phase 2: Implement Controller (2-3 hours)

```typescript
// api/controllers/GameController.ts
export async function getPlayerGameState(context: ExegesisContext) {
    const { gameId, playerId } = context.params.path;
    
    const game = await store.read<Game>(store.keys.games, gameId);
    const world = await store.read<World>(store.keys.worlds, game.worldId);
    
    // Apply visibility filter
    const filteredWorld = await filterWorldForPlayer(world, playerId);
    
    return {
        gameId: game.id,
        playerId,
        turn: game.turn,
        world: filteredWorld,
        playerCount: game.players.length
    };
}
```

### Phase 3: Update Client (1-2 hours)

Add to API client and use in GameView component.

### Phase 4: Testing (1-2 hours)

Test visibility filtering and response format.
